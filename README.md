# プログラミング言語

だいたい初めて触った順に書いてます

|言語|好き度|コメント|
|---|---|---|
|F-BASIC386|★★★|中学生のとき初めて触った言語。ゲームを作る楽しさを教えてくれた。~だが知ってる人に会ったことがない~ Unityギルドで知っている人に出会った|
|C|★★★|大学生の時に勉強し始め。プログラマになること目指し本格的に勉強始めた第一歩。手続き型言語って結局なんなんだろう|
|C++|★★★|**今までで最も多く書いている言語。** Cの1年後くらいに始め。コンシューマゲーム(業務用,PS3,3DS,PSVita)、スマホゲーム(cocos2d-x)と経て10年以上、いまだに使ってる。今振り返ればこれを最初に覚えたおかげでその後のどの言語も習得が容易だった|
|CommonLISP|★☆☆|学生の頃授業でやったきり。appendとlistとconsの違いは理解した。10年の時を経てRxの理解を助ける事になる|
|アセンブラ|★★☆|授業で初めて触った時、なんの処理系だったかはもはや覚えてないが、BEQやBNEの後ろにNOP書くのは無駄だから一つ前の命令を書くべし、というテクに感銘を受けた|
|Java|★☆☆|卒論はこれで書いた。その後Androidのゲーム・アプリを作るようになり長い付き合いとなる(でも好きではない)|
|JavaScript|★☆☆|いつ始めたかは記憶にない。なんだかんだちょいちょい書いているが、実務でがっつり使ったことはない|
|bash|★★☆|書けるけど本当に必要な時だけ書きます。pecoがお気に入り|
|Perl|★★★|DeNAとの仕事の時に、ソシャゲのバックエンドを書いた。Perl自体を使う期間は長くなかったが(1年ほど)、短く書けるため大好きだった|
|MySQL|★★☆|トーゼン書けます|
|Ruby|★★★|Railsで主にバックエンドで使用。ソシャゲでもWebサービスでも使ってます|
|Objective-C|★★☆|iPhone3GSの時代にアプリを作るため書き始めた。ゲーム本体はC++で書くため、Objective-Cで書くのはプラットフォーム依存の部分くらい|
|Swift|★★☆|propertyWrapperとか自動アンラップとか、コードの本質に集中できる書き方ができて好きです。ただSwiftUIの癖がまだ慣れない|
|C#|★★★|Unityとともに2011年ごろから使い出した。UniRxが登場してからガラッと書き方が変わった。言語の進化も速いし今一番好きです|
|Solidity|★★☆|ブロックチェーンに興味を持ち、CryptZombieのおかげで書けるようになった。でも実務で書く機会がない。gasコストを減らすためにアセンブラレベルで考えるのは好き。こんなところでアセンブラと再会するとは思ってなかった|
|PHP|★☆☆|1ヶ月だけ実務で使ったことがある。FuelPHPだった|
|Rust|★★☆|C/C++をやってきた者としてはコンパイラの優秀さにとにかく感銘。関数の戻り時に所有権が移動する素晴らしさ|
|Go|★☆☆|良い言語だと思うけど、Goが活かせるプロダクトに出会えてないから実感として持ててないところあり|
|TypeScript|★★☆|型っていいよね。仕様が複雑というけどわかってくると便利だなと思うところが多い|
|Cadence|★★☆|Flow blockchainのコントラクトを書くために使用。リソースの線形管理は納得度高いです|

### fingger (2021/2~6)
- ゲーム配信プラットフォーム
  - UnityWebGLで作られたゲームを配信
  - 動画配信者はゲームをプレイし、このプラットフォームとYouTubeで配信、視聴コメントが可能
  - 投げ銭が開発者にも分配されるサービスです
- Unity/C#/dotnet
  - ゲームはUnityなのでC#、サーバサイドもdotnetでC#で合わせていました
  - ゲームとバックエンドが両方わかるということで、ゲームとバックエンド間のリアルタイム通信を担当しました
    - リアルタイム通信はMagicOnionを利用（しかも今プロジェクトのためにWebGL版を新規で作っていただいたものでした）
  - そのほかYouTubeとの同期（コメント情報など）も実装しました（だいぶYouTubeAPIに詳しくなりました）
